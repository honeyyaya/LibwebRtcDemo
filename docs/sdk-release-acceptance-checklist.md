# SDK Release Acceptance Checklist

适用范围：面向外部交付的 `libRoboFlow Client SDK`。

使用方式：
- 发布前由研发、测试、集成方共同过表。
- 每项标记为 `Pass` / `Fail` / `N/A`。
- `必须项` 不能有 `Fail`，否则不得发布。
- `应该项` 允许少量延期，但必须记录风险和补救计划。
- `可延期项` 可进入后续版本，但要明确版本目标。

## 必须项

### 1. 接口与 ABI

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 头文件声明与库导出符号完全一致 | 不允许出现“头文件有、so 里没有” |
| [ ] | 所有公开函数名、参数、返回值与文档一致 | 包括命名、语义、错误码 |
| [ ] | 已发布函数签名未发生破坏性变更 | 只允许新增，不允许修改旧签名 |
| [ ] | 枚举值只追加、不复用 | 保证老版本调用方兼容 |
| [ ] | 导出符号经过白名单或版本脚本控制 | 避免内部符号误导出 |
| [ ] | 版本号可查询，且与发布包一致 | `get_version` / `get_build_info` 可用 |

### 2. 生命周期主链路

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 最小链路可跑通 | `set_global_config -> init -> connect -> open_stream -> on_video -> close_stream -> disconnect -> uninit` |
| [ ] | `init/uninit` 幂等行为符合文档 | 重复调用结果可预测 |
| [ ] | `connect/disconnect` 幂等行为符合文档 | 重复调用不崩溃、不泄漏 |
| [ ] | `open_stream/close_stream` 幂等行为符合文档 | 包含重复开同一路、重复关流 |
| [ ] | 失败后可重新初始化并再次连接 | 不要求重启进程 |
| [ ] | `disconnect/uninit` 后无悬挂线程和资源泄漏 | 包括 socket、媒体对象、回调对象 |

### 3. 线程与回调模型

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 回调线程模型与文档一致 | 不能“文档一套、实现一套” |
| [ ] | 同一回调对象串行性与文档一致 | connect_cb / stream_cb |
| [ ] | 回调内允许调用的 API 经实测可用 | 如 getter、retain/release、close_stream |
| [ ] | 回调 drain 语义符合文档 | `disconnect/uninit` 返回后行为稳定 |
| [ ] | `userdata` 生命周期约束清晰且实测无 UAF | 集成方可安全释放 |

### 4. 错误处理

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 每个同步 API 失败时返回稳定错误码 | 不返回模糊成功 |
| [ ] | 异步错误通过回调 `reason` 稳定上报 | 不要求调用方猜测 |
| [ ] | `last_error` 语义与文档一致 | 仅补充同步错误诊断 |
| [ ] | 不存在“返回 RFLOW_OK 但实际未执行”的假成功 | 尤其是 notice / reply / stats / connect |
| [ ] | 错误信息足够定位问题 | 至少覆盖参数、状态、网络、能力不支持 |

### 5. 功能完成度

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 所有公开导出的 API 都已真实实现 | 不允许只声明不实现 |
| [ ] | 所有公开能力都已打通 | 包括 stats、notice、service reply 等 |
| [ ] | `connect` 语义与文档一致 | 鉴权、连接、设备语义不可误导 |
| [ ] | `open_stream` 状态机与文档一致 | OPENING / OPENED / FAILED / CLOSED |
| [ ] | `on_video` 数据格式、生命周期、retain/release 语义可被调用方正确使用 | 不靠隐式约定 |

### 6. 打包与交付

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 发布包包含完整头文件、库文件、示例、版本说明 | 调用方可独立接入 |
| [ ] | 不缺三方依赖或运行时依赖说明 | Android/Linux/Windows 分平台说明清楚 |
| [ ] | 示例工程可直接编译并跑通 | 不是“仅供参考、不可运行” |
| [ ] | Debug/Release 产物边界清晰 | 文件名、符号、日志策略明确 |

## 应该项

### 7. 可观测性

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 支持日志等级设置 | 运行期可调 |
| [ ] | 支持日志回调 | 方便集成方接入自身日志系统 |
| [ ] | 关键状态迁移有日志 | init/connect/open/close/fail |
| [ ] | 关键失败点带上下文信息 | 如 stream index、device id、signal url |
| [ ] | 发布包附常见错误码说明 | 降低集成成本 |

### 8. 配置能力

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 全局配置默认值明确 | 未配置时行为可预测 |
| [ ] | 配置生效时机明确 | init 前、connect 前、运行期 |
| [ ] | 非必填配置缺省行为有文档 | 如 license、signal、log |
| [ ] | 配置错误会明确失败 | 不静默降级 |
| [ ] | 关键网络参数可控 | timeout、重连、keepalive 等 |

### 9. 测试覆盖

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 单元测试覆盖公开 API 基本行为 | create/destroy、参数校验、错误码 |
| [ ] | 集成测试覆盖完整生命周期 | 最小链路必须自动化 |
| [ ] | 覆盖重复调用与幂等测试 | 重复 init/connect/open/close |
| [ ] | 覆盖异常路径 | 超时、断网、非法参数、服务端异常 |
| [ ] | 覆盖资源泄漏检查 | 内存、线程、句柄 |
| [ ] | 覆盖符号导出检查 | 防止下次再出 undefined symbol |

### 10. 文档质量

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 有 Quick Start 文档 | 调用方 30 分钟内能跑通 |
| [ ] | 每个 API 都有线程、生命周期、所有权说明 | 不是只有函数声明 |
| [ ] | 有升级说明和变更日志 | 说明兼容性影响 |
| [ ] | 有 FAQ | 常见错误、回调线程、释放时机 |
| [ ] | 文档与当前发布版本一致 | 不引用未来能力 |

## 可延期项

### 11. 增强能力

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 完整 stats 拉取与周期推送 | 若主链路已稳定，可后续增强 |
| [ ] | 更细粒度 trace / profiling 能力 | 面向性能分析 |
| [ ] | 更丰富的 signal / control 面能力 | 例如 topic、扩展 notice |
| [ ] | 多路流高级能力 | 当前若只支持主路可后续扩展 |
| [ ] | 更完整的跨平台工具链验证 | 如更多 ABI / 架构组合 |

### 12. 工程化增强

| 状态 | 验收项 | 说明 |
|---|---|---|
| [ ] | 自动生成 API 参考文档 | 例如 Doxygen / Sphinx |
| [ ] | 自动 ABI 对比报告 | 版本发布时自动出结果 |
| [ ] | 发布包签名与校验文件 | SHA256 / 签名 |
| [ ] | 示例工程矩阵 CI | Android / Linux / Windows 全覆盖 |
